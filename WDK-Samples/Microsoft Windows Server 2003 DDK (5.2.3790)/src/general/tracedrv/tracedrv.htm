Tracedrv (Windows XP Sample)

 

SUMMARY

 

TraceDrv is an example of a very simple driver which really does nothing. It is intended purely as a code sample to show the use of Software Tracing in a driver.

 

The driver initializes tracing and on receipt of a DeviceIOControl it will start a thread that logs 100 trace messages.

 

The changes connected with software tracing are commented, and this example is discussed in the DDK documentation.

 

Note: This sample driver is not a PnP driver. This is a minimal driver meant to demonstrate an OS feature. Neither it nor its sample program is intended for use in a production environment. Rather, they are intended for educational purposes and as a skeleton driver.

 

TESTING

 

To test this driver, run TraceCTL.exe, a simple Win32 console application. TraceCTL will automatically load the driver and start it. Once the driver is started, TraceCTL will send an IOCTL to the driver and the driver will start a thread to log 100 trace messages.
